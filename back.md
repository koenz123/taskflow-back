# Спецификация бэкенда Telephone Bot (для Cursor / разработчика бэкенда)

Скопируй этот файл в чат Cursor в репозитории бэкенда и попроси реализовать описанные маршруты и логику. Бэкенд развёрнут на том же сервере, что и фронт Taskflow; фронт вызывает эти эндпоинты по тому же origin (или по `VITE_API_BASE`).

---

## 1. Назначение

Голосовой ассистент для входящих звонков через **Telphin (Teleo)**:

- **Call Interactive** — вебхук, который Telphin вызывает при звонке. Бэкенд получает распознанный текст абонента (STT), отправляет его в LLM (OpenAI или GigaChat), возвращает **XML** с озвучкой ответа (TTS) по формату Telphin.
- **Тестовый диалог** — REST endpoint для проверки без звонка: фронт отправляет текст, бэкенд возвращает ответ нейросети и время отклика.

---

## 2. Маршруты (обязательные)

### 2.1 Call Interactive (вебхук Telphin)

- **Методы:** `GET` и `POST`
- **Путь:** `/telphin/call-interactive`
- **Вход:** параметры от Telphin в query (GET) или в body (POST). Ключевой параметр:
  - `voice_navigator_STT` — распознанный текст абонента (при первом запросе может отсутствовать — тогда отдать приветствие).
- **Выход:** ответ с `Content-Type: application/xml` — XML по документации Telphin Call Interactive (см. ссылку ниже). В XML — озвучка ответа бота (TTS). При необходимости перехода обратно в голосовое меню использовать значения из env: `IVR_JUMP_CONTEXT`, `IVR_JUMP_OPTION`.

**Логика:**

- Если есть `voice_navigator_STT` — отправить текст в LLM, получить ответ, сформировать XML с озвучкой этого ответа.
- Если текста нет (первый запрос в сценарии) — вернуть XML с приветствием (например из системного промпта или захардкоженное «Здравствуйте, чем могу помочь?»).
- Документация формата ответа: [Telphin Call Interactive](https://ringme-confluence.atlassian.net/wiki/spaces/Ringme/pages/1901920992).

### 2.2 Тест диалога (без звонка)

- **Метод:** `POST`
- **Путь:** `/api/telephone/test-dialog`  
  (фронт вызывает именно этот путь: `API_BASE + '/telephone/test-dialog'`, где `API_BASE` обычно `/api`.)
- **Заголовки:** `Content-Type: application/json`
- **Тело запроса:** `{ "userText": string }`
- **Ответ:** `200 OK`, JSON: `{ "reply": string, "elapsedMs"?: number }`
  - `reply` — текст ответа нейросети.
  - `elapsedMs` — время обработки в миллисекундах (опционально, но желательно).
- **Ошибки:** при ошибке (нет ключа LLM, таймаут, и т.д.) — ответ с кодом 4xx/5xx и JSON `{ "error": string }`.

**Логика:** вызвать тот же сервис/LLM, что и для Call Interactive (без генерации XML), вернуть только текст ответа и время.

## 3. Переменные окружения (.env)

| Переменная | Обязательно | Описание |
|------------|-------------|----------|
| `OPENAI_API_KEY` | один из двух | Ключ OpenAI. Если задан — используется ChatGPT. |
| `OPENAI_MODEL` | нет | Модель OpenAI (по умолчанию `gpt-4o-mini`). |
| `GIGACHAT_CREDENTIALS` | один из двух | Ключ GigaChat (Base64). Используется, если `OPENAI_API_KEY` не задан. |
| `GIGACHAT_SCOPE` | нет | Scope GigaChat (например `GIGACHAT_API_B2B`). |
| `GIGACHAT_MODEL` | нет | Модель GigaChat (по умолчанию можно `GigaChat-2-lite`). |
| `PORT` | нет | Порт сервера (по умолчанию 3000). |
| `APP_BASE_URL` | нет | Публичный URL бэкенда (для отображения вебхука и для Telphin). URL вебхука: `{APP_BASE_URL}/telphin/call-interactive`. |
| `SYSTEM_PROMPT` | нет | Системный промпт ассистента (например краткое описание бота). |
| `IVR_JUMP_CONTEXT` | нет | Контекст IVR для перехода обратно в голосовое меню (из настроек Teleo). |
| `IVR_JUMP_OPTION` | нет | Опция IVR для перехода (из настроек Teleo). |
| `CORS_ORIGIN` | по необходимости | Origin фронта (если фронт на том же сервере — можно задать origin или `*`). |

## 4. Архитектура бэкенда

- **Слои:** controller → service → (при необходимости) репозиторий. В контроллере только приём запроса и отдача ответа; бизнес-логика и формирование XML — в сервисе.
- **Модули:**
  - Клиент(ы) LLM: OpenAI, GigaChat (выбор по наличию `OPENAI_API_KEY`).
  - Сервис диалога: приветствие, вызов LLM, формирование текста ответа.
  - Формирование XML для Telphin по документации Call Interactive (отдельная функция/модуль).
- Валидация входа (например `userText` не пустой для test-dialog). Обработка ошибок LLM и возврат понятного `error` в JSON.

---

## 5. Согласование с фронтом

- Фронт вызывает **POST** `{API_BASE}/telephone/test-dialog` с телом `{ userText: string }`. Итоговый путь при `API_BASE = '/api'`: `/api/telephone/test-dialog`.
- URL вебхука для Telphin, который показывается пользователю на экране: **`{APP_BASE_URL}/telphin/call-interactive`** (значение `APP_BASE_URL` из .env — публичный адрес бэкенда без слэша в конце). Маршрут Call Interactive доступен по пути `/telphin/call-interactive` (без префикса `/api`), так как Telphin дергает «корневой» URL сервера.

Если твой бэкенд отдаёт API только под префиксом `/api`, нужно либо вынести `GET/POST /telphin/call-interactive` на корень (отдельный роут без префикса), либо документировать итоговый URL и поправить фронт (показ webhook URL).

## 6. Краткий чеклист реализации

- [ ] Роут `GET` и `POST` `/telphin/call-interactive`: парсинг параметров (в т.ч. `voice_navigator_STT`), вызов сервиса диалога, возврат XML.
- [ ] Роут `POST` `/api/telephone/test-dialog`: body `{ userText }`, вызов LLM, ответ `{ reply, elapsedMs }`.
- [ ] Подключение OpenAI и/или GigaChat по env.
- [ ] Формирование XML для Telphin по документации Call Interactive.
- [ ] Переменные окружения из раздела 3.
- [ ] CORS разрешает запросы с фронта (тот же origin или заданный `CORS_ORIGIN`).

После реализации бэкенд будет готов к работе с экраном «Телефонный бот» во фронте Taskflow и с настройкой IVR в личном кабинете Teleo.
